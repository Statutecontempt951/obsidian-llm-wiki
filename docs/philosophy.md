# vault-mind 设计哲学

> 知识不是被管理的，是自己运动的。

## 三个理论基座

### 矛盾论 → Agent Scheduler

事物发展的根本原因在事物内部的矛盾性。vault 的自运动不靠外力（cron/用户调用），靠内在矛盾。

| 概念 | 映射 |
|------|------|
| 主要矛盾 | evaluate() 找当前 vault 最阻碍知识发展的矛盾 |
| 次要矛盾 | 排队等主要矛盾解决后升级 |
| 矛盾转化 | 主要矛盾解决 -> 次要矛盾升级为新主要矛盾 -> 持续运动 |
| 对抗性矛盾 | 两个 concept 直接冲突 -> 必须调和 |
| 非对抗性矛盾 | 视角不同但不冲突 -> 综合为更高层次 |
| 矛盾的特殊性 | 每个 vault 的主要矛盾不同 |

### 实践论 → 编译循环

通过实践发现真理，通过实践证实和发展真理。

```
感性认识 (raw notes)
    -> 理性认识 (compiled concepts)
    -> 实践 (用知识做决策/写代码/交易)
    -> 结果写回 vault
    -> 新的感性认识
    -> cycle
```

实践是检验真理的唯一标准：concept 指导的决策失败 -> 写回 vault -> 触发重编译 -> 理性认识被修正。

### 群众路线 → 知识的民主集中制

从群众中来，到群众中去。

```
散落笔记 (群众)
    -> compile + emerge (集中)
    -> feed back to daily notes, skills, decisions (到群众中去)
    -> 日常使用产生新笔记 (再从群众中来)
```

## 核心设计原则

1. **矛盾驱动，不是清单驱动** -- agent 不遍历优先级列表，分析主要矛盾
2. **编译是飞跃，不是整理** -- 从感性到理性的质变，不是换个格式存
3. **实践检验，不是自洽即真** -- concept 必须被实践验证，失败的 concept 必须被修正
4. **群众路线，不是精英设计** -- 散落笔记是知识的源泉，不是待清理的垃圾
5. **内在运动，不是外力维护** -- vault 的自组织靠矛盾推动，不靠用户勤奋
